Мета: придбати практичні навички роботи  реалізації методів перевантаження унарних, бінарних, логічних операторів, операторів порівняння.

Постановка задачі:

  1. Доповнити програмну реалізацію класів методами для перевантаження унарних, бінарних, логічних операторів, операторів порівняння.

  1.   Протестувати програму. Зробити висновки.

Визначимо оператори порівняння для класу Score. Це сутність що відображає кількість забитих та пропущених голів. Тому ми можемо застосвувати оператори порівняння для 2 сутностей для виконання спеціальної визначеної логіки.

В даному випадку оператори унарні та бінарні додавання будуть сумувати кількість забитих та пропущених голів для 2 сутностей з класу Score. Виконання логічних операцій дозволить порівнювати швидко дві сутності таблиці Score зручно застосувавши лише оператори <, <=, ==, !=, >=,>.

Висновок: в ході виконання даної лаборатоної роботи я придбала практичні навички роботи реалізації методів перевантаження унарних, бінарних, логічних операторів, операторів порівняння.

Іноді порівняння двох об'єктів не так просто, як порівняння двох чисел - для цього C # дозволяє писати свої власні, призначені для користувача методи для виконання таких речей, як порівняння. C # надає можливість будувати спеціальні класи і структури, які також унікально реагують на один і той ж набір базових лексем (на кшталт операції +)

Перевантаження оператора в основному є механізмом надання особливого значення звичайному логічному оператору C # для визначеного користувачем типу даних, таких як структури або класи.

Причиною того, що розробники зазвичай додають оператори до власних типів або класів, є забезпечення сумісності з операторами, які вже працюють із вбудованими типами. Інша причина полягає в тому, що, можливо, ви захочете надати подібну функціональність вбудованим операторам.
